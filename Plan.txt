**Automated Pull Request Description Generator using Fine-Tuned Language Models**

---

# **1. Problem Statement**

In modern software development, pull requests (PRs) are essential for collaboration and code review. However, developers often write vague or incomplete PR descriptions such as *“minor changes”* or *“bug fix”*, which:

- Reduce code readability and maintainability
- Make code reviews inefficient
- Create poor long-term documentation

Although platforms like GitHub provide basic commit suggestions, they lack deep semantic understanding of code changes.

👉 Therefore, there is a need for an intelligent system that can automatically generate **clear, meaningful, and structured PR descriptions** from code changes.

---

# **2. Literature Review**

Recent advancements in **Generative AI and code understanding models** have enabled automatic code summarization and documentation.

- Research such as *“Sphinx: Benchmarking and Modeling for LLM-Driven Pull Request Review (2026)”* explores how large language models can interpret code diffs and assist in PR reviews.
- Studies on commit message generation show that traditional models struggle with capturing **developer intent**.
- Models like CodeLlama and DeepSeek-Coder demonstrate strong capabilities in code understanding but require **task-specific fine-tuning** for structured outputs like PR descriptions.

👉 Existing systems:

- Focus on simple summarization
- Lack fine-grained intent understanding
- Do not enforce structured PR documentation

---

# **3. Objectives**

The main objectives of this project are:

1. To design a system that understands **code diffs (before vs after changes)**
2. To generate **human-like PR titles and descriptions**
3. To improve **code documentation quality**
4. To apply **fine-tuning techniques (LoRA/QLoRA)** on a pre-trained model
5. To evaluate generated outputs using standard NLP metrics

---

# **4. Proposed Solution**

This project proposes a **fine-tuned generative AI model** that converts code changes into meaningful PR descriptions.

---

## 🔄 Workflow

```
Git Diff (input)
        ↓
Preprocessing (cleaning + formatting)
        ↓
Fine-tuned Language Model
        ↓
Generated Output:
   • PR Title
   • PR Description
   • Change Type (optional)
```

---

## 🧠 Key Idea

The model learns a mapping:

```
Code Changes → Intent → Natural Language Explanation
```

---

## ✨ Example

**Input (diff):**

```
- password = "1234"
+ password = hashPassword(userInput)
```

**Output:**

> “Improved security by replacing hardcoded password with hashed user input.”

# **5. Methodology**

### 1. Data Collection

- Extract **(diff, PR description)** pairs from datasets

### 2. Data Preprocessing

- Clean diff format
- Remove noise
- Normalize text

### 3. Model Selection

- Base model: small code LLM (7B or smaller)

### 4. Fine-Tuning Approach

- Use **LoRA / QLoRA (Parameter Efficient Fine-Tuning)**
- Train model on domain-specific mapping

### 5. Evaluation

- Compare:
    - Base model (few-shot prompting)
    - Fine-tuned model
